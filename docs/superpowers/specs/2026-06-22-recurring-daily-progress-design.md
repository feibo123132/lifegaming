# 日常任务累计进度徽标设计

## 目标

为“每天自动出现”的日常任务显示累计进度徽标，例如截至 2026-06-22，当任务从 2026-06-09 开始、过去 13 次中完成 11 次时显示 `11/13`。

## 计数口径

- 使用 `templateId` 关联同一个日常任务的所有实例，避免不同任务之间串计数。
- 过去日期的实例均计入分母，不论完成或失败。
- 今天的实例仅在已经完成时计入分子和分母；尚未完成时不计入，因为当天还没有结束。
- 未来实例不计入。
- 只奖不罚的重复日常任务沿用相同进度口径。
- 周期挑战保留自身的 `已完成天数/目标天数` 逻辑；非重复日常任务不显示累计徽标。

## 展示

在任务卡片的“每天”徽标后追加黄色累计进度徽标，文本为 `已完成次数/已结算次数`，不追加“天”字。

## 测试

领域测试覆盖：过去完成与失败的混合记录、今天未完成不计入、今天完成后立即计入、未来任务排除，以及不同 `templateId` 之间互不影响。
