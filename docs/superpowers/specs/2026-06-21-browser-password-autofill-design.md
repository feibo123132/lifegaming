# 本机浏览器账号密码自动填充设计

## 目标

当 CloudBase 会话失效或用户回到登录页时，登录页自动进入密码登录模式并带出最近成功登录的邮箱；密码由当前电脑上的 Chrome 或 Edge 密码管理器安全填充。应用、`localStorage`、仓库和云端均不保存密码。

## 现状与参考项目结论

- 参考项目使用 `persistence: "local"` 保持 CloudBase 会话，并通过标准 `autocomplete` 属性让浏览器识别账号密码表单；没有把密码写入 `localStorage`。
- 当前项目已经使用 CloudBase 本地会话持久化、认证状态监听和 `RequireAuth` 路由守卫。会话失效时，认证状态会变为未登录并返回登录页。
- 当前密码输入框带有 `autocomplete="current-password"`，但邮箱输入框位于密码表单之外、缺少稳定的 `name` 和账号型 `autocomplete`，浏览器不一定能把两者识别为同一组凭据。
- 当前项目支持通过 Vite 环境变量把默认密码注入前端。这会把密码编译进浏览器可读取的前端产物，不符合“仅在自己的电脑安全保存”的目标。

## 设计

### 最近登录邮箱

- 沿用现有 `design_life_user_email` 本地键，但将其语义明确为“最近成功登录邮箱”。
- 只有成功登录并取得 CloudBase 用户信息时才更新该值。
- 会话自动失效或用户主动退出时不删除该邮箱；它只是登录便利信息，不代表当前仍处于登录状态。
- 登录页优先使用开发环境配置的默认邮箱，否则使用最近登录邮箱。

### 浏览器密码管理器

- 邮箱输入框使用稳定的 `name="username"` 和 `autocomplete="username"`。
- 密码输入框使用稳定的 `name="password"` 和 `autocomplete="current-password"`。
- 通过 HTML `form` 属性把页面上共享的邮箱输入框关联到密码登录表单，让浏览器将邮箱和密码识别为同一组凭据。
- 登录页存在已记住邮箱时默认打开密码登录模式，使密码框在掉线返回登录页时立即挂载并允许浏览器填充。
- 应用不自动提交登录，避免无人操作时直接进入账号。

### 密码安全

- 移除 `VITE_DEFAULT_LOGIN_PASSWORD` 的读取和示例配置。
- `.env.example` 不包含真实账号或密码，仅保留空白邮箱示例和安全说明。
- 不新增任何密码存储键，也不把密码传给认证接口之外的逻辑。

### 数据保护

- 现有游戏数据先写入 Zustand 本地持久化，再尝试云端同步；会话失效不会抹掉本地快照。
- 重新登录同一邮箱后，现有同步逻辑会复用并合并本地较新的数据。
- 本功能不改变跨账号隔离规则：若改用其他邮箱登录，不会把上一账号的本地数据合并过去。

## 验证

- 单元测试覆盖默认邮箱、最近邮箱回退、初始登录模式和本地邮箱读写。
- 认证相关现有测试继续通过。
- TypeScript 类型检查通过。
- 生产构建尝试一次；若出现仓库说明中的 `esbuild spawn EPERM`，按环境限制止损。
