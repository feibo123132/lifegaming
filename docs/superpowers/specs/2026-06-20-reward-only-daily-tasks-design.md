# “只奖不罚”每日任务设计

## 目标

让“锻炼”一类鼓励型习惯每天自动出现：完成时正常增加积分和经验；未完成时不扣积分、不扣经验，也不影响完成率或失败复盘。

## 交互设计

- “每天自动出现”继续只负责重复生成每日任务。
- 日常任务新增独立开关“只奖不罚”。它可以与“每天自动出现”同时启用。
- “周期挑战”保持独立的特殊模式；开启周期挑战时关闭“每天自动出现”和“只奖不罚”，避免互相冲突。
- 只奖不罚任务在任务卡片上显示“只奖不罚”标签，让用户能识别其规则。

推荐创建“锻炼”的配置为：勾选“每天自动出现”和“只奖不罚”，不勾选“周期挑战”。

## 数据与计算规则

- `Task` 和 `DailyTaskTemplate` 增加可选布尔字段 `rewardOnly`；缺省值视为 `false`，兼容已有本地及云端数据。
- 每日模板生成任务实例时继承 `rewardOnly`。
- 只奖不罚任务完成后仍按任务积分正常奖励，并可贡献连续打卡。
- 未完成的只奖不罚任务不属于逾期任务，因此积分、经验和失败复盘均不受影响。
- 完成率统计始终排除只奖不罚任务，无论它是否完成，避免改变统计分母或产生超过 100% 的结果。
- 首页任务完成数和任务页日/周/月统计使用同一套完成率计算规则。

## 验证

- 领域测试覆盖创建、每日模板继承、逾期免罚、完成奖励、完成率排除及失败复盘排除。
- 运行现有 `gameSync` 测试、TypeScript 类型检查和生产构建，确认没有回归。
